# Excel导入格式说明

本文档说明学生管理、用户管理、院系管理三个模块的Excel导入格式要求。

---

## 一、学生管理Excel导入格式

### 支持的字段
- **学号**（可选，但学号和姓名至少需要填写一个）
- **姓名**（可选，但学号和姓名至少需要填写一个）
- **类型**（可选，支持"论文"、"设计"或"PAPER"、"DESIGN"）
- **题目**（可选）
- **所属院系**（可选，支持院系名称或院系代码）

### 格式要求

#### 方式一：带表头（推荐）
第一行为表头，从第二行开始为数据。

| 学号 | 姓名 | 类型 | 题目 | 所属院系 |
|------|------|------|------|----------|
| 20210001 | 张三 | 论文 | 基于深度学习的图像识别研究 | 计算机科学与技术学院 |
| 20210002 | 李四 | 设计 | 智能家居控制系统设计 | CS |
| 20210003 | 王五 | PAPER | 区块链技术在金融领域的应用 | 软件学院 |

**表头识别规则：**
- 学号列：包含"学号"、"studentNo"、"student_no"
- 姓名列：包含"姓名"、"name"
- 类型列：包含"类型"、"type"、"defenseType"
- 题目列：包含"题目"、"title"
- 所属院系列：包含"所属院系"、"院系"、"department"、"dept"

#### 方式二：无表头
从第一行开始就是数据，按固定顺序：
- 第1列：学号
- 第2列：姓名
- 第3列：类型
- 第4列：题目
- 第5列：所属院系

**示例：**
```
20210001	张三	论文	基于深度学习的图像识别研究	计算机科学与技术学院
20210002	李四	设计	智能家居控制系统设计	CS
20210003	王五	PAPER	区块链技术在金融领域的应用	软件学院
```

### 注意事项
1. **学号和姓名至少需要填写一个**：如果学号为空，系统会自动生成一个基于姓名和时间戳的学号；如果姓名为空，系统会自动生成一个基于学号的姓名。
2. **类型值**：支持中文"论文"、"设计"或英文"PAPER"、"DESIGN"，不区分大小写。
3. **每行至少有一个字段有数据**：学号、姓名、类型、题目、所属院系中至少需要填写一个。
4. **学号唯一性**：同一答辩年份内，学号不能重复。
5. **所属院系**：支持院系名称（如"计算机科学与技术学院"）或院系代码（如"CS"）。如果未填写，系统会使用当前登录用户的院系（院系管理员）或为空（超级管理员）。
6. **其他字段**：指导教师、评阅教师等字段会在导入后由管理员手动分配。

### 导入示例文件结构
```
学号        姓名    类型    题目                            所属院系
20210001    张三    论文    基于深度学习的图像识别研究      计算机科学与技术学院
20210002    李四    设计    智能家居控制系统设计            CS
20210003    王五    论文    区块链技术在金融领域的应用      软件学院
```

## 二、用户管理Excel导入格式

### 支持的字段
- **用户名**（可选，如果未提供会自动生成）
- **真实姓名**（可选）
- **角色**（可选，支持角色名称或中文名称）
- **院系**（可选，支持院系代码或院系名称）
- **状态**（可选，支持0/1或"启用"/"禁用"，默认为1-启用）

### 格式要求

#### 方式一：带表头（推荐）
第一行为表头，从第二行开始为数据。

| 用户名 | 真实姓名 | 角色 | 院系 | 状态 |
|--------|----------|------|------|------|
| admin001 | 张三 | SUPER_ADMIN | CS | 1 |
| dept_admin | 李四 | 院系管理员 | 计算机科学与技术学院 | 启用 |
| T001 | 王五 | TEACHER | CS | 1 |

**注意**：教师角色的用户名就是教师编号（如示例中的T001），系统会自动在教师表中创建对应的教师记录。

**表头识别规则：**
- 用户名列：包含"用户名"、"username"、"user"
- 真实姓名列：包含"真实姓名"、"姓名"、"realname"、"real_name"、"name"
- 角色列：包含"角色"、"role"
- 院系列：包含"院系"、"department"、"dept"
- 状态列：包含"状态"、"status"

#### 方式二：无表头
从第一行开始就是数据，按固定顺序：
- 第1列：用户名
- 第2列：真实姓名
- 第3列：角色
- 第4列：院系
- 第5列：状态

**示例：**
```
admin001	张三	SUPER_ADMIN	CS	1
dept_admin	李四	院系管理员	计算机科学与技术学院	启用
T001	王五	TEACHER	CS	1
```

**注意**：教师角色的用户名就是教师编号（如示例中的T001），系统会自动在教师表中创建对应的教师记录。

### 角色值说明
支持以下角色名称（不区分大小写）：
- `SUPER_ADMIN` 或 "超级管理员"
- `DEPT_ADMIN` 或 "院系管理员"
- `TEACHER` 或 "教师"
- `DEFENSE_LEADER` 或 "答辩组长"

### 状态值说明
- `1` 或 "启用" 或 "激活" 或 "enabled" 或 "active" → 启用
- `0` 或 "禁用" 或 "停用" 或 "disabled" 或 "inactive" → 禁用
- 默认为 1（启用）

### 注意事项
1. **每行至少有一个字段有数据**：用户名、真实姓名、角色、院系、状态中至少需要填写一个。
2. **用户名唯一性**：如果提供了用户名，系统会检查是否已存在，如果存在则导入失败。
3. **自动生成用户名**：如果未提供用户名，系统会根据真实姓名自动生成，格式为：`真实姓名_时间戳`。
4. **默认密码**：新导入的用户默认密码为"123456"（已加密），建议用户首次登录后修改密码。
5. **角色验证**：如果提供了角色，系统会验证角色是否存在，不存在则导入失败。
6. **院系验证**：如果提供了院系，系统会通过院系代码或名称查找，找不到则导入失败。
7. **教师角色特殊说明**：
   - 如果角色是"TEACHER"（教师），**用户名就是教师编号**（teacher_no）
   - 教师角色必须提供用户名（作为教师编号），否则导入失败
   - 导入教师用户时，系统会自动在教师表（teacher）中创建对应的教师记录
   - 教师表的teacher_no字段 = 用户表的username字段
   - 教师表和用户表通过user_id字段关联

### 导入示例文件结构
```
用户名        真实姓名    角色            院系                    状态
admin001      张三        SUPER_ADMIN     CS                      1
dept_admin    李四        院系管理员      计算机科学与技术学院     启用
T001          王五        TEACHER         CS                      1
```

**注意**：教师角色的用户名就是教师编号（如示例中的T001），系统会自动在教师表中创建对应的教师记录。

## 三、院系管理Excel导入格式

### 支持的字段
- **ID**（可选）
- **院系名称**（可选）
- **院系代码**（可选）
- **描述**（可选）

### 格式要求

#### 方式一：带表头（推荐）
第一行为表头，从第二行开始为数据。

| ID | 院系名称 | 院系代码 | 描述 |
|----|----------|----------|------|
| 1 | 计算机科学与技术学院 | CS | 计算机相关专业 |
| 2 | 软件学院 | SE | 软件工程相关专业 |
| 3 | 信息与通信工程学院 | ICE | 通信工程相关专业 |

**表头识别规则：**
- ID列：包含"ID"、"id"
- 院系名称列：包含"院系名称"、"名称"、"name"
- 院系代码列：包含"院系代码"、"代码"、"code"
- 描述列：包含"描述"、"description"、"desc"

#### 方式二：无表头
从第一行开始就是数据，按固定顺序：
- 第1列：ID
- 第2列：院系名称
- 第3列：院系代码
- 第4列：描述

**示例：**
```
1	计算机科学与技术学院	CS	计算机相关专业
2	软件学院	SE	软件工程相关专业
3	信息与通信工程学院	ICE	通信工程相关专业
```

### 注意事项
1. **每行至少有一个字段有数据**：ID、院系名称、院系代码、描述中至少需要填写一个。
2. **ID唯一性**：如果提供了ID，系统会检查是否已存在，如果存在则导入失败。
3. **院系代码唯一性**：如果提供了院系代码，系统会检查是否已存在，如果存在则导入失败。
4. **其他字段**：未填写的字段会保持为空（NULL）。

### 导入示例文件结构
```
ID    院系名称              院系代码    描述
1     计算机科学与技术学院   CS         计算机相关专业
2     软件学院               SE         软件工程相关专业
3     信息与通信工程学院     ICE        通信工程相关专业
```

## 通用说明

### 文件格式
- 支持 `.xlsx`（Excel 2007及以上版本）
- 支持 `.xls`（Excel 2003版本）

### 数据验证
- 所有导入都会进行数据验证
- 如果部分数据导入失败，会显示详细的错误信息，包括失败行号和原因
- 导入结果会显示成功和失败的统计信息

### 权限要求
- **学生导入**：需要院系管理员或超级管理员权限
- **用户导入**：需要超级管理员权限
- **院系导入**：需要超级管理员或院系管理员权限

### 常见错误
1. **列名不匹配**：确保表头使用正确的列名，或使用无表头格式
2. **数据类型错误**：确保数字列输入的是数字，文本列输入的是文本
3. **必填字段为空**：确保必填字段（如学生的学号或姓名）有值
4. **唯一性冲突**：确保唯一字段（如学号、用户名、院系代码）不重复
5. **关联数据不存在**：确保关联数据（如角色、院系）在系统中已存在
